# Runbooks

Use this folder to store operational runbooks that make the capstone feel like a real production platform.

Recommended runbooks:
- failed deployment rollback
- pod crash investigation
- node pool failure response
- secret rotation
- TLS certificate issue response
- database restore procedure
- high error rate alert response
- cluster access recovery

Each runbook should include:
- trigger condition
- immediate checks
- probable causes
- remediation steps
- rollback or escalation path
- post-incident notes
